# 44.5-01a — Visual Context System reference index

Research deliverable for task 44.5-01a. Produced 2026-04-15 as the "what references the Project Editor needs to consume" input to the 44.5-01 KICKOFF.md. Not a full plan; a reference index.

**Scope rule (from operator, 2026-04-15)**: the Visual Context System (VCS) is mandatory for every visual system — web pages, games, editors, modals, dashboards, any graphical surface. Debug-mode UI built on VCS is the foundation for all future brownfield work. Any new GUI surface must include VCS infrastructure before feature work starts. The two anchor tokens the operator flagged by name:

- `cid="visual-context-modal-footer"` — the compact strip variant shown inside portaled modals.
- `stableCid="visual-context-panel"` — the full floating dev panel chrome shown on regular pages.

Both are source-greppable literals defined in constants files and referenced via the `Identified` / `DevPanel` components below.

## CRUD semantics the operator flagged (2026-04-15)

These behaviors MUST be preserved on every new editor chrome, not just copied visually:

| Action | Current shape | Operator framing |
|---|---|---|
| **Copy cid** (grab search literal) | `navigator.clipboard.writeText(current.searchHint ?? current.cid)` in `DevIdModalContextFooter.copyCid`, line 210. Alt+click anywhere on an `Identified` child does the same via `Identified.handleClick`, line 104–114. | "mainly copy to clipboard for easy grab of context" |
| **UPDATE** prompt | `buildUpdateLockedPrefix` emits locked markdown prefix → prepended to editable body → `navigator.clipboard.writeText(resolved)` in `DevPanelHoverPromptActions.finalizeUpdate`, line 66–81. Optionally fed from speech transcript via `useDictatedPromptCopy`. | "quickly capture with light prompts, speech capture and prompt injecting" |
| **DELETE** prompt | `buildDeletePrompt` emits a deterministic delete instruction block → `navigator.clipboard.writeText(resolved)` in `DevPanelHoverPromptActions.copyDelete`, line 97–110. No editable body — fire-and-forget copy. | "delete is almost always a simple copy to clipboard" |
| **Speech capture** | Mic button toggles `speechRecognition.ts` Web Speech API → transcript accumulates → on finalize runs through `buildUpdateLockedPrefix` → copied to clipboard. | "speech capture and prompt injecting" |
| **Agent Message** | `DevIdAgentPromptDialog` full editor opens on click, lets operator refine before copying. For high-friction edits where speech + quick prompt aren't enough. | (implied by the hover action set) |

**Invariant**: every CRUD action on VCS surfaces is implemented via clipboard copy. The operator pastes into their coding agent. VCS does NOT invoke agents directly from the page — the page produces a clipboard payload with source search literals, and the operator moves it to the agent they choose (Claude Code, Cursor, Codex, etc). This preserves operator choice of agent and keeps the dev surface non-destructive.

## What is already available for the Project Editor to consume vs. what needs to be built

### Already available (zero new infra)

- `Identified`, `SiteSection`, `DevIdProvider`, `SectionRegistry` — every new editor pane can wrap its content in `<SiteSection cid="project-editor-<pane>-site-section">` or `<Identified as="ProjectEditor<Pane>" cid="project-editor-<pane>">` and automatically participates in the existing band dev panel.
- `DevPanel` + `BandDevPanel` — will auto-mount for any `SiteSection` added to the Project Editor route; no wiring.
- `DevIdModalContextFooter` — can be embedded inside any modal the editor opens (inventory-grid slot-edit modal, recipe-edit dialog, species-json field modal) by passing `scanRootRef` pointing at the modal root.
- `DevPanelHoverPromptActions` — copy-to-clipboard UPDATE/DELETE + speech capture already generic, no changes needed.
- `DevIdPromptTemplates.ts` — prompt builders already parameterized on component tag / label / cid / searchHint, no changes needed.
- `DevIdSearchDialog` — global search already indexes every `data-cid` in the page regardless of route.
- `useDictatedPromptCopy` — speech capture hook already generic.

### Needs to be built specifically for the Project Editor (and new games/apps)

1. **New stable cids + constants file** for each top-level editor chrome that is not a `SiteSection`:
   - Project Editor shell → `project-editor-shell-site-section` (just a SiteSection, no constants file needed).
   - Split-viewport left pane → `project-editor-bestiary-pane-site-section` (SiteSection).
   - Split-viewport right pane (iframe preview) → `project-editor-preview-pane-site-section` (SiteSection).
   - Bottom inspector pane → `project-editor-inspector-pane-site-section` (SiteSection).
   - Inventory-grid slot modal → new constants file: `site/components/project-editor/inventory-grid-constants.ts` exporting `INVENTORY_GRID_MODAL_SELF_ENTRY` with `cid="project-editor-inventory-grid-modal"`.
   - Recipe-edit dialog → similar constants file.
   - Species-json field edit modal → similar constants file.
   - Each modal must also embed a `DevIdModalContextFooter` scanning its own root.
2. **Deterministic cid derivation** for minecraft inventory grid slots (task 44.5-05 dependency): each slot must derive its cid from the config key path, e.g. `inventory-slot-gad-json-sprintSize-site-section`. This is new infrastructure: a helper `buildInventorySlotCid(configPath: string[])` that lives alongside the inventory grid primitive.
3. **Deterministic cid derivation for bestiary cards** (task 44.5-07): each bestiary card derives its cid from species slug, e.g. `bestiary-card-<project>-<species>-site-section`. Each generation row inside the card derives its cid as `bestiary-generation-<project>-<species>-v<N>-site-section`.
4. **Deterministic cid derivation for diff tree rows** (task 44.5-09): each diff-row derives its cid from the config key path being diffed.
5. **iframe postMessage bridge for inner-build cids** (task 44.5-10 stretch goal): when the iframe-previewed build opts in via a postMessage handshake, the outer editor's modal footer should be able to target cids inside the iframe. Requires an inner-build opt-in contract. Not mandatory for 44.5-10 first cut — fallback is that the iframe wrapper is the single grep-able target.
6. **VCS coverage regression test** — a test that walks every page under `/projects/[id]/edit/**` in rendered form and asserts every `role="region"` / `role="dialog"` / major visual block has a `data-cid` attribute. Prevents new editor work from drifting. Candidate location: `vendor/get-anything-done/tests/vcs-coverage.test.cjs` or a Playwright assertion.

## Summary for the 44.5-01 KICKOFF.md

- 14 devid components + 1 SiteSection component + 2 constants files + 2 skills + 4 decisions + 1 memory entry = the full VCS reference surface.
- Every new editor chrome must EITHER be a `SiteSection` (free VCS participation) OR be wrapped in `Identified` + ship its own constants file analogous to `dev-modal-footer-constants.ts` + embed a `DevIdModalContextFooter` if it opens a modal.
