# Design moderation rules for champion-title + leaderboards

**Source:** session 2026-04-17 strategy pivot

Per decisions gad-248 + gad-249 + phase 54.

Before implementing the token transfer + leaderboard surfaces, design moderation rules.

## Questions to answer

- Who can report a species / fork / generation?
- What actions can moderators take (freeze transfer, remove contributor, etc.)?
- How does appeal work?
- What auto-triggers a freeze (sudden contribution spike, coordinated activity)?
- How do we surface moderation history publicly vs keep private?
- Do we need moderator roles tiered (community mod / platform mod / admin)?
- Rate limits on token transfers to prevent churn?

## Reference

Lean on existing moderation patterns from game leaderboards + code hosting (GitHub abuse reports) rather than inventing from scratch.
