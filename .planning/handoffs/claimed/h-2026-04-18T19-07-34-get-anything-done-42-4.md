## Top 3 cuts (ranked by saving / breakage risk)

### 1. Drop \`cancelled\` tasks from the TASKS section (15 rows ≈ 300 tokens)

Current snapshot shows 15 cancelled tasks from phases 20, 21, 22. These are permanent history — not actionable, not blocking, not consulted. Agents pick them out of context when looking for 'planned' work.

**Fix:** `bin/gad.cjs` — find the TASKS section builder (grep the literal \`-- TASKS (\`). Filter out \`status=\"cancelled\"\` unless \`--full\` or \`--include-cancelled\` flag is passed. Keep the \"54 open, 373 done\" header accurate.

**Test:** `tests/snapshot-filters.test.cjs` — assert no \`[cancelled]\` rows in default output, presence with `--include-cancelled`.

**Saving:** ~300 tokens per call.

### 2. Limit TASKS to current-sprint phases by default (~300 tokens)

Snapshot currently prints tasks from phases 20, 21, 22, 35, 42.2, 42.3, 43, 44, 44.5, 45, 60, 63 — 12 phases. Sprint 9 per the roadmap header is only: 42.4, 43, 44, 44.5, 45. Out-of-sprint phases add noise.

**Fix:** default task listing to current-sprint phases (already computed for the ROADMAP header). Out-of-sprint tasks visible via `gad tasks --projectid <id>`. Preserve phase 60 + phase 63 by promoting them to the active sprint when ready (operator-controlled via `gad sprint` CLI).

**Saving:** ~300 tokens per call.

### 3. XML entity decode in task goals (~40 tokens + readability)

Task goals contain visible `&lt;` / `&gt;` / `&apos;` that should be decoded when rendering to text. Example (real): \`skills/*/SKILL.md description for overlapping trigger keyword\` — the goal actually says `<gt>` literally. Other entries show `evals/&lt;project&gt;/gad.json`.

**Fix:** snapshot rendering path runs each text field through a `decodeXmlEntities(s)` helper (`&amp;` → `&`, `&lt;` → `<`, `&gt;` → `>`, `&apos;` → `'`, `&quot;` → `\"`). Small pure function. Probably lives in a render/format util next to the snapshot builder.

**Saving:** small bytes + agent readability (current output looks broken).
