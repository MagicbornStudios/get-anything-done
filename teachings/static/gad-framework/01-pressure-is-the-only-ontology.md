# Pressure is the only ontology

GAD's north-star principle, formalized as decision gad-250:

> **Nothing exists in a project, species, or generation until pressure is produced.**

A project with 500 decisions, 2000 commits, 200 tasks, and ZERO pressure produced against its core problem has produced nothing. It has generated motion, not realization.

## What pressure actually means

Pressure is a GAD-specific formula (see `references/pressure-formula.md`) that measures how hard the project has pushed against the real problem it claims to solve. It's a function of:

| Input | Signal |
|---|---|
| Friction encountered | Real work meeting real resistance |
| Commitments kept | Promises the system made and delivered on |
| Cycles closed | Loops that actually completed, not abandoned |
| Feedback absorbed | External signal processed, not deflected |

A project that only does the easy parts, never closes loops, and avoids feedback produces low pressure regardless of how much time was spent.

## Why gauges multiply by pressure

GAD tracks four gauges (decision gad-251):

| Gauge | What it measures |
|---|---|
| Decisions | Direction — how sharply has the project committed? |
| Errors-and-attempts | Stability — how many times has the same problem bit, and has it been solved? |
| Notes | Knowledge density — how much durable understanding has accumulated? |
| Task throughput | Realized output — planned vs completed per sprint |

Each gauge is **multiplied** by the pressure score, not added to it. Pressure is the gate. Zero pressure = zero realization, no matter how pretty the other numbers look.

## The practical test

Before declaring a project "productive," ask:

1. What real friction did we encounter this week?
2. What loop did we close that we couldn't have closed last week?
3. What feedback did we absorb that changed our direction?

If the answers are "none / none / none" but the commit count is high, you produced motion, not pressure. Motion is addictive. Pressure is the only thing that matters.

## Takeaway

Realization is not an accumulation of activity. It's a function of sustained friction against the real problem. When your gauges light up but nothing feels harder than yesterday, you're probably measuring motion.
