# Always Measure Token Costs
Every prompt, cache entry, and context snapshot should log its byte and token count as early as possible — usable context starts with knowing exactly what it costs.

Knowing the token cost for each context component is the single best lever on context efficiency. It’s almost impossible to spot waste, optimize caching, or budget snapshots if you can’t see what costs what, in real numbers. For instance, OpenAI’s GPT-4 Turbo charges $0.01 per 1,000 prompt tokens and $0.03 per 1,000 sampled tokens as of 2024 — so a 3,000-token mistake can balloon into a $0.09 invoice just to process the wrong code example.

Here’s what you get with and without accurate measurement:

| Approach                       | Observability        | Surprise Costs | Intent Alignment | Engineering Friction |
|-------------------------------|---------------------|---------------|------------------|---------------------|
| Measure token/byte counts      | Precise, auditable  | Never         | Always           | Low (can automate)  |
| Estimate, guess, or ignore     | Fuzzy, lagging      | Frequently    | Rarely           | High (debug pain)   |

**Example:**  
If you cache a user’s last 8 code edits in a snapshot, and each is ~400 bytes formatted, the snapshot is 3.2 KB. On GPT-4 Turbo, this is about 900 tokens (at ~3.5 characters/token). But code samples compress sharply if canonicalized (e.g., stripping comments, dedenting), and so a 25% reduction is routine for heavily-indented Python — that’s 225 tokens saved per snapshot, or $2.50 per 100,000 snapshots.

**How to implement:**  
Instrument every prompt-construction function to log:
- Raw text length in bytes
- Token count using your model’s tokenizer package (`tiktoken` for OpenAI, `tokenizers` for HF models, etc.)
- Costs at current rates (dynamic, not hardcoded)

For OpenAI models as of 2024, a simple `tiktoken` call in Python:
```python
import tiktoken
enc = tiktoken.encoding_for_model("gpt-4-turbo")
tokens = len(enc.encode(prompt_text))
print(f"{tokens} tokens, ${(tokens / 1000) * 0.01:.5f} USD prompt cost")
```

**Operational discipline:**  
Log token costs in CI runs, production logs, and even local dev tooling. Surface top context cost outliers in dashboards and merge requests, similar to test or performance regressions.

| Tool/Method        | Supported Models    | Byte Count | Token Count | Typical Integration      |
|--------------------|--------------------|------------|------------|-------------------------|
| tiktoken           | OpenAI, GPT-4, etc.| Yes        | Yes        | Python, CLI, Caching    |
| tokenizers (HF)    | Llama, Mistral     | Yes        | Yes        | Rust, JS, Python        |
| OpenAI API         | All                | No         | Yes        | Auto, after request     |

## Takeaway

If you’re not measuring prompt and snapshot token costs at every code and context boundary, you’re flying blind and wasting both context and dollars.
